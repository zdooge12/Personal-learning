Breadcrumb Secondary Navigation

Display and Position
  • Breadcrumbs are usually displayed as a horizontal list of pages and take up minimal space.
  • Users expect to find them in the header, left-aligned, and below any primary navigation.
  • Typically they are separated with a " > " or a "/" symbol.
  • In the case of a lengthy breadcrumb sequence, use "..." to annotate ommissions
Common Breadcrumb Marking Conventions
  • .breadcrumb li+li::before is an example of a selector to manage content we want to display before list items
  • This code can be modified with the content property set to, for example " > " to place the greater sign between each adjacent breadcrumb.
  • Another feature of breadcrumbs is that they are often not underlined. Underlines are used to communicate that text within paragraph elements can be clicked, but we want our users to recognize breadcrumbs as part of the site navigation, rather than just like any other clickable text.
  • This is done with the css property: text-decoration: none;
  • Breadcrumbs should change when you hover over them to highlight their clickability. Commonly this is done with the color property using the a:hover pseudo-selector
